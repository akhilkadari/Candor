# Contributing

Candor is currently in hackathon submission mode. The immediate goal is stability, clarity, and demo readiness rather than broad external feature intake.

## Current Expectations

- Keep changes tightly scoped.
- Prefer documentation, bug fixes, and demo-readiness improvements over broad refactors.
- Preserve the privacy-first and on-device nature of the product.
- Avoid changing core behavior casually in the recovery flow without validating the user experience impact.

## Contribution Priorities

- Build reliability
- Documentation quality
- Recovery flow polish
- Insight generation correctness
- Demo stability on Android hardware

## Before Opening A PR

1. Build from `Android/src`.
2. Verify the recovery flow still works end to end.
3. Confirm documentation stays aligned with implementation.
4. Keep unrelated local IDE files out of commits.

## At This Stage

If this repository is being evaluated for the hackathon, treat the docs as part of the product surface. Changes that improve readability, setup clarity, and evaluation flow are high value.
